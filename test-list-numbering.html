🔧 列表编号重复问题已修复

解决了有序列表渲染时出现重复编号的bug

🐛 问题描述

之前在渲染Notion页面时，当有序列表被分批加载或分段显示时，会出现编号重复的问题。例如：

修复前（错误）

  1. 第一项内容
  2. 第二项内容

-- 页面分批加载后 --

  1. 重复的编号2（应该是3）
  2. 重复的编号3（应该是4）

修复后（正确）

  1. 第一项内容
  2. 第二项内容

-- 页面分批加载后 --

  1. 正确的编号3
  2. 正确的编号4

🔧 技术修复

我们在以下三个关键函数中修复了编号计算逻辑：

1. renderBlocks() 函数

+ let orderedListStart = 1; // 追踪当前有序列表的起始编号
+ if (listTag === 'ol') {
+ content += `<ol start="${orderedListStart}">`;
+ orderedListStart++;
+ }

2. renderIncrementalBlocks() 函数

+ const orderedListStart = calculateOrderedListStart(pageContent, loadingIndicator);
+ processedContent += `<ol start="${orderedListStart}">`;

3. renderFinalBatch() 函数

+ if (listTag === 'ol') {
+ const orderedListStart = calculateOrderedListStart(pageContent, loadingIndicator);
+ processedContent += `<ol start="${orderedListStart}">`;
+ }

✅ 修复效果

  • 编号连续性：确保有序列表在分批加载时编号连续
  • 上下文感知：自动检测前一个列表的结束编号
  • 智能重置：非列表内容会重置编号计数器
  • 性能优化：避免了不必要的DOM重计算

🧪 如何测试

  1. 访问包含有序列表的Notion页面
  2. 确保页面内容较长，会触发分批加载
  3. 观察列表编号是否连续
  4. 确认不再出现重复编号的问题！
